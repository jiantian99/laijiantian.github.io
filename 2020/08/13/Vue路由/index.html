Vue Router

Vue Router是Vue.js官方的路由管理器（路径跳转）。它和Vue.js的核心深度集成，让构建单页面应用变得易如反掌。包含的功能有:

  1. 嵌套的路由/视图表
  2. 模块化的、基于组件的路由配置
  3. 路由参数、查询、通配符
  4. 基于Vue.js过渡系统的视图过渡效果
  5. 细粒度的导航控制
  6. 带有自动激活的CSS class的链接
  7. HTML5历史模式或hash模式，在IE9中自动降级
  8. 自定义的滚动条行为

安装

npm install vue-router --save-dev

实例

index.html调用main.js，再调用App.vue

index.html

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Vue</title>
</head>
<body>
<div id="app"></div>
</body>
</html>

main.js

import Vue from 'vue'
import App from './App'

// 自动扫描路由配置
import router from './router'

Vue.config.productionTip = false

/* eslint-disable no-new */
new Vue({
  el: '#app',
  // 配置路由
  router,
  components: { App },
  template: '<App/>'
})

App.vue

  • 使用<router-link>组件来导航，传入to属性指定链接，<router-link>默认渲染成一个<a>标签

<template>
  <div id="app">
    <h1>使用Vue Router</h1>
    <router-link to="/test">Test</router-link>
    <router-link to="/main">Main</router-link>
    <router-link to="/foo">Foo</router-link>
    <router-view></router-view>
  </div>
</template>

<script>

export default {
  name: 'App',
}
</script>

路由配置 index.js (默认路由配置文件名)

// 导入Vue和VueRouter
import Vue from 'vue'
import VueRouter from "vue-router";
// 导入组件
import Test from "../components/Test";
import Main from "../components/Main";
import Foo from "../components/Foo";
// 安装路由
Vue.use(VueRouter)

// 配置导出路由
export default new VueRouter({
  routes:[
    // Test 组件
    {
      // 路由的路径
      path: '/test',
      name: 'content',
      // 跳转的组件
      component: Test
    },
    {
      // 路由的路径
      path: '/main',
      name: 'content',
      // 跳转的组件
      component: Main
    },
    {
      // 路由的路径
      path: '/foo',
      // name: 'content',
      // 跳转的组件
      component: Foo
    },
  ]
})

嵌套路由

嵌套路由又称子路由，在实际应用中，通常由多层嵌套的组件组合而成。同样地，URL 中各段动态路径也按某种结构对应嵌套的各层组件，例如：

/user/foo/profile                     /user/foo/posts
+------------------+                  +-----------------+
| User             |                  | User            |
| +--------------+ |                  | +-------------+ |
| | Profile      | |  +------------>  | | Lists       | |
| |              | |                  | |             | |
| +--------------+ |                  | +-------------+ |
+------------------+                  +-----------------+

在VueRouter参数中使用children配置嵌套路由

routes:[
    {
      path: '/main',
      component: Main,
      // 导入子模块
      children:[
        {
          path: '/user/profile',
          component: UserProfile,
        },{
          path: '/user/list',
          component: UserList,
        }
      ]
    }
]

参数传递

当请求时，需要传递参数

  • 第一种取值方式，使用$route.params

    1. 修改路由配置index.js

      {
    path: '/user/profile/:id',
    name: 'UserProfile',
    component: UserProfile,
}

    2. 在<route-link>处改成:to，将这一属性当作对象来使用，<route-link>中的name要与路由配置中的name匹配

      <router-link :to="{name:'UserProfile',params:{id:1}}">个人信息</router-link>

    3. 在展示的组件Profile.vue中接收参数

      {{$route.params.id}}

  • 第二种取值方式，使用props减少耦合

    1. 路由配置index.js中增加props: true属性

      {
    path: '/user/profile/:id',
    name: 'UserProfile',
    component: UserProfile,
    props: true
}

    2. 在展示的组件Profile.vue中接收参数，增加props参数

      <template>
  <div>
    <h1>个人信息</h1>
    {{id}}
  </div>
</template>

<script>
  export default {
    props: ['id'],
    name: "UserProfile"
  }
</script>

重定向和别名

重定向也是通过 routes路由 配置来完成

  • 从/goHome到/main

    {
    path: '/goHome',
	redirect: '/main'
}

  • 重定向的目标也可以是一个有命名的路由

    {
	path: '/goHome',
	redirect: {name: 'main'}
}

重定向指的是：访问/goHome，url替换成/main，路由匹配的是/main

别名指的是：/main的别名是/goHome，访问/goHome时，url保持/goHome，但路由匹配的是/main，就像直接访问/main一样

{
	path: '/main',
	name: 'main',
    component: Main,
    alias: '/goHome',
}

路由模式

  • hash：路径带 # 符号（默认），如 http://localhost/#/login
  • history：路径不带 # 符号，如 http://localhost/login
export default new VueRouter({
    mode: 'history',
    routes: [
    ]
});

404

使用通配符*，并放在路由配置的最后

{
    path: '*',
    component: NotFound
}
