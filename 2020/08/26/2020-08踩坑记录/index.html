在Centos8系统编译Mysql5.7.27时,出现依赖包不全的情况报错

安装所需依赖包，并重新预编译。重新预编译前，需要删除”CMakeCache.txt”文件

Package ‘libtirpc‘, required by ‘virtual:world‘, not found

报错内容：

-- Checking for module ‘libtirpc‘
--   Package ‘libtirpc‘, required by ‘virtual:world‘, not found
CMake Error at cmake/rpc.cmake:65 (MESSAGE):
  Could not find rpc/rpc.h in /usr/include or /usr/include/tirpc
Call Stack (most recent call first):
  plugin/group_replication/libmysqlgcs/configure.cmake:57 (MYSQL_CHECK_RPC)
  plugin/group_replication/libmysqlgcs/CMakeLists.txt:28 (INCLUDE)

解决方法：

yum install -y libtirpc-devel

Could not find rpcgen

报错内容：

CMake Error at rapid/plugin/group_replication/rpcgen.cmake:93 (MESSAGE):
  Could not find rpcgen
Call Stack (most recent call first):
  rapid/plugin/group_replication/CMakeLists.txt:29 (INCLUDE)

解决方法：

wget https://github.com/thkukuk/rpcsvc-proto/releases/download/v1.4/rpcsvc-proto-1.4.tar.gz
tar xf rpcsvc-proto-1.4.tar.gz
cd rpcsvc-proto-1.4
./configure
make
make install

Spring Boot 设置拦截器，报空指针异常

报错内容：在登录拦截器中，redisPool报了空指针异常

java.lang.NullPointerException: null
// 错误的方法
@Override
public void addInterceptors(InterceptorRegistry registry) {
    // 增加登录拦截器
    registry.addInterceptor(new LoginHandlerInterceptor()).addPathPatterns("/obs/**");
}

分析：在配置登录拦截器时，使用了new 对象的方法，正确方式应该是使用Spring的依赖注入，让Spring容器帮我去new一个登录拦截器对象，并检查所需的依赖，自动装配。

解决方法：

@Autowired
LoginHandlerInterceptor loginHandlerInterceptor;
@Override
public void addInterceptors(InterceptorRegistry registry) {
    // 增加登录拦截器
    registry.addInterceptor(loginHandlerInterceptor).addPathPatterns("/obs/**");
}
